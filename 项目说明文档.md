# 企业邮箱售卖网站项目说明文档

## 项目概述

本项目是一个企业邮箱售卖网站，提供企业邮箱账号的在线购买、支付和自助提取功能。网站使用 Django 框架开发，部署在 Linux 服务器上，通过 Nginx 和 Gunicorn 提供服务。

## 项目技术栈

- **前端**：HTML5, CSS3, JavaScript, Bootstrap
- **后端**：Python 3.12, Django 6.0.4
- **服务器**：Nginx, Gunicorn
- **数据库**：SQLite
- **版本控制**：Git, GitHub

## 项目结构

```
/var/www/docstore/
├── config/            # Django 配置文件
├── store/             # 业务逻辑应用
├── static/            # 静态文件
├── staticfiles/       # 静态文件集合
├── templates/         # HTML 模板
├── db.sqlite3         # SQLite 数据库
├── manage.py          # Django 管理脚本
├── requirements.txt   # 项目依赖
└── venv/              # Python 虚拟环境
```

## 核心功能

1. **邮箱套餐展示**：展示不同类型的企业邮箱套餐
2. **在线购买**：用户可以选择套餐并完成支付
3. **自助提取**：支付成功后，用户可以自助提取邮箱账号和密码
4. **代理合作**：提供代理注册和管理功能
5. **后台管理**：管理员可以管理邮箱套餐、订单和用户
6. **库存管理**：支持批量导入邮箱账号，自动识别两种格式：
   - 单条账号：账号----密码----密钥 或 账号----密码
   - 组账号：以 ------域名_数字-------- 作为分组标记，每组最多50个账号，每组第一个为主账号
   - 导入规则：如果每组账号数量小于或等于46个，去除主账号加入按条售卖；只有每组47个以上才加入组售卖

## 部署配置

### Nginx 配置

Nginx 作为反向代理服务器，将请求转发到 Gunicorn 处理动态内容，并直接提供静态文件服务。

### Gunicorn 配置

Gunicorn 作为 WSGI 服务器，运行 Django 应用，处理动态请求。

### 环境变量

项目使用以下环境变量：
- `DEBUG`：调试模式开关
- `SECRET_KEY`：Django 密钥
- `ALLOWED_HOSTS`：允许访问的域名列表
- `CSRF_TRUSTED_ORIGINS`：信任的 CSRF 来源
- `SITE_BASE_URL`：网站基础 URL
- `EMAIL_*`：邮件服务器配置
- `CKKP_*`：支付网关配置
