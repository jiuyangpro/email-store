# 企业邮箱售卖网站错误集合文档

## 错误列表

### 1. SSH 连接失败（退出码 255）

**错误描述**：SSH 连接服务器时出现退出码 255 错误。

**原因**：沙盒环境限制，无法创建 `.ssh` 目录（Permission denied）。

**解决方案**：
- 手动输入密码进行连接
- 避免使用需要创建 `.ssh` 目录的操作

**预防措施**：
- 在正常环境中使用 SSH 密钥认证，避免密码输入
- 确保 `.ssh` 目录存在且权限正确

### 2. `sshpass` 命令未找到

**错误描述**：执行 `sshpass` 命令时提示未找到该命令。

**原因**：当前环境未安装 `sshpass` 工具。

**解决方案**：
- 放弃使用 `sshpass`，改用手动输入密码
- 在支持的环境中安装 `sshpass`：`apt install sshpass`（Ubuntu/Debian）

**预防措施**：
- 提前检查环境中是否安装了所需工具
- 准备备用方案，避免依赖特定工具

### 3. PowerShell 不支持 `&&` 操作符

**错误描述**：在 PowerShell 中执行多命令时使用 `&&` 导致语法错误。

**原因**：PowerShell 不支持 `&&` 操作符，需要使用分号 `;` 分隔命令。

**解决方案**：
- 将多命令用分号 `;` 分隔，如 `cd D:\脚本项目\服务器项目\app; git add .`

**预防措施**：
- 了解不同终端环境的命令语法差异
- 编写跨平台兼容的命令

### 4. 网站无法访问（服务不可用）

**错误描述**：访问 `http://172.245.196.85/` 时提示服务不可用。

**原因**：
- Nginx 配置未指向 Django 应用，默认服务静态文件
- Django 应用未启动

**解决方案**：
- 修改 Nginx 配置文件，将请求代理到 Gunicorn（127.0.0.1:8000）
- 在服务器上启动 Gunicorn 并重启 Nginx

**预防措施**：
- 确保 Nginx 配置正确指向后端服务
- 确保后端服务（Gunicorn）正常运行

### 5. Git 提交时中文乱码

**错误描述**：Git 提交信息中的中文显示乱码。

**原因**：终端编码问题导致中文显示乱码。

**解决方案**：
- 不影响功能，提交成功后在 GitHub 上显示正常
- 可设置终端编码为 UTF-8：`chcp 65001`（Windows PowerShell）

**预防措施**：
- 确保终端编码设置为 UTF-8
- 使用英文提交信息避免编码问题

### 6. Internal Server Error

**错误描述**：访问网站时出现 500 内部服务器错误。

**原因**：
- Gunicorn 服务未运行
- Nginx 配置错误
- 端口 8000 被占用

**解决方案**：
- 创建并启动 Gunicorn 系统服务
- 修复 Nginx 配置文件中的静态文件路径
- 终止占用端口 8000 的进程
- 重新启动 Gunicorn 服务

**预防措施**：
- 确保 Gunicorn 服务设置为开机自启
- 定期检查服务运行状态
- 避免端口冲突

### 7. 400 错误（Bad Request）

**错误描述**：访问网站时出现 400 错误，提示 "Invalid HTTP_HOST header"。

**原因**：域名不在 Django 的 `ALLOWED_HOSTS` 列表中。

**解决方案**：
- 修改 Django 配置文件 `settings.py`，将域名添加到 `ALLOWED_HOSTS` 列表
- 重启 Gunicorn 服务以应用新配置

**预防措施**：
- 在部署前确保所有访问域名都已添加到 `ALLOWED_HOSTS` 列表
- 使用环境变量配置 `ALLOWED_HOSTS`，方便不同环境部署

### 8. 静态文件加载失败

**错误描述**：网站只能看到文字，没有样式和图片，界面显示不完整。

**原因**：Nginx 配置文件中的静态文件路径错误，导致静态资源（CSS、JavaScript、图片等）无法加载。

**解决方案**：
- 找到实际生效的 Nginx 配置文件（如 `/etc/nginx/sites-available/docstore`）
- 修改静态文件路径，将错误的路径（如 `/var/www/docstore/app/staticfiles/`）改为正确的路径（如 `/var/www/docstore/staticfiles/`）
- 重启 Nginx 服务以应用新配置

**预防措施**：
- 确保 Nginx 配置文件中的静态文件路径与 Django 的 `STATIC_ROOT` 配置一致
- 执行 `collectstatic` 命令后，验证静态文件是否正确收集到 `STATIC_ROOT` 目录
- 定期检查 Nginx 配置文件，确保路径配置正确

### 9. 后台管理页面选项显示问题

**错误描述**：在后台管理页面中，售卖类型和 2FA 状态选项无法正常显示，选择后不显示选中的选项。

**原因**：`PackageAdminForm` 类的 `__init__` 方法中缺少对 `twofa_status` 字段的初始化，导致表单字段无法正确显示。

**解决方案**：
- 修改 `admin.py` 文件，在 `PackageAdminForm` 类的 `__init__` 方法中添加对 `twofa_status` 字段的初始化
- 添加字段标签：`self.fields["twofa_status"].label = "2FA状态"`
- 添加字段帮助文本：`self.fields["twofa_status"].help_text = "选择商品的 2FA 状态"`
- 重启 Gunicorn 服务以应用新配置

**预防措施**：
- 在添加新字段到模型后，确保在相应的表单类中也添加对该字段的初始化
- 测试表单字段的显示和交互，确保所有字段都能正确显示和操作
- 定期检查后台管理页面的表单配置，确保所有字段都能正常显示

## 错误处理最佳实践

1. **详细记录错误**：遇到错误时，详细记录错误信息、发生时间和上下文
2. **分析根本原因**：不仅解决表面问题，还要分析根本原因，避免类似错误再次发生
3. **制定预防措施**：针对每个错误，制定相应的预防措施
4. **定期更新文档**：每次解决错误后，及时更新错误集合文档
5. **建立监控机制**：设置服务监控，及时发现和处理错误
